Первоначальную настройку SmartXML рекомендуется производить с использованием графического интерфейса. Однако в целях упрощения автоматизации обработки приложение может быть запущено вовсе без загрузки графического интерфейса.

  • Открыть проект с указанным именем:

    .\SmartXML -proj=sample_1

  • Открыть проект с указанным именем и установить номер Job:

    .\SmartXML -proj=sample_1 -job=2

    Примечание: указание номера Job важно только в случае если XML для обработки содержатся в таблице БД

    Более подробная информация доступна на странице jobs-settings

    Если Job не указан, то проект будет открыт по умолчанию с Job равным 1.

  • Открыть проект с указанным именем, установить номер Job и автоматически запустить парсинг:

    .\SmartXML -proj=sample_1 -job=2 -start

    Примечание: указание номера Job важно только в случае если XML для обработки содержатся в таблице БД.

    Более подробная информация доступна на странице jobs-settings.

    Если Job не указан, то проект будет открыт по умолчанию с Job равным 1.

    Ключ -start используется исключительно при работе с базой данных и НЕ используется при обработке одиночных файлов.

  • Не запуская графический интерфейс открыть проект с указанным именем, установить номер Job и автоматически запустить парсинг:

    .\SmartXML -proj=sample_1 -job=2 -start -noui

    Примечание: указание номера Job важно только в случае если XML для обработки содержатся в таблице БД.

    Более подробная информация доступна на странице jobs-settings.

    Если Job не указан, то проект будет открыт по умолчанию с Job равным 1.

    Ключ -start используется исключительно при работе с базой данных и НЕ используется при обработке одиночных файлов.

    По завершению всех задач парсинга, а так же если нет XML файлов для обработки выполняется выход из приложения.

  • Запустить проект sample_1 и подставить XML файл для обработки. Ждать нажатия кнопки в интерфейсе:

    .\SmartXML.exe -proj=sample_1 -input=D:/code/SmartXML/data-samples/sample-1/example_1.xml

  • Запустить проект sample_1, подставить XML для обработки, указать папку в которую нужно записать результат и целевой формат. Не отображать интерфейс:

    .\SmartXML.exe -proj=sample_1 -input=D:/code/SmartXML/data-samples/sample-1/example_1.xml -output=D:\test\output -format=json -noui

    Примечание: целевая папка должна существовать.

    Поддерживаются формат json и sql.

  • Запустить проект sample_1, подставить XML для обработки, записать результат в тот же каталог где расположен оригинальный XML, указать целевой формат. Не отображать интерфейс:

    .\SmartXML.exe -proj=sample_1 -input=D:/code/SmartXML/data-samples/sample-1/example_1.xml -output=D:\test\output -format=json -noui

    Поддерживаются формат json и sql.

  • Следующие теги можно комбинировать с примерами выше:

    По умолчанию парсер останавливается встретив неизвестные теги. Следующие аргументы командной строки позволяют изменить это поведение:

    .\SmartXML.exe -lu

    Логировать неизвестные теги. Добавлять их в интерфейс если он загружен:

    .\SmartXML.exe -su

    Пропускать неизвестные теги. Не останавливаться на них и не добавлять их в интерфейс (если он загружен).

    Примечание: Данные ключи могут потребоваться если в процессе пакетной обработки в XML документах могут встретиться неизвестные теги, обработка которых не принципиальна, и нет причин останавливать процесс обработки, а сами эти теги достаточно залогировать и/или пропустить.

Если вам нужны гарантии того, что процесс будет перезапущен даже в случае возможной ошибки вы можете воспользоваться следующим bat файлом: 

                        
                            @echo off
                            :Start
                            D:\code\SmartXML\SmartXML.exe -proj=name -job=1 -start
                            :: Wait 30 seconds before restarting.
                            TIMEOUT /T 30
                            GOTO:Start
